Several functions from the kotlinx.coroutines library such as filterNotNull() or filterIsInstance have sense only when they are called on receivers that have types distinct from the resulting one. Otherwise, such calls can be omitted as the result will be the same.

Remove redundant call quick-fix can be used to amend the code automatically.

Example:


  fun test(flow: Flow<String>) {
      val x = flow.filterNotNull() // quick-fix simplifies to 'flow'
      val y = flow.filterIsInstance<String>() // quick-fix simplifies to 'flow'
  }
